    Domain Driven Design

    This example focusses on how to do Domain Driven Design throught the `z-dragon` tool. Types are simple definitions of the "shape" of data. An aggregate is something which is where everything comes together. Here we can define the shape but also the behavior of the types.

    Person Aggregate

    Operation on the Person aggregate:

    Name Input Output Description
    getPerson
    id: String
    		 PersonAggregate
    updatePerson
    PersonAggregate: PersonAggregate
    		 PersonAggregate
    updatePersonAddress
    Address: Address
    		 PersonAggregate
    deletePersonAggregate
    id: String
    		 Boolean description: Purge a person from the repository.
